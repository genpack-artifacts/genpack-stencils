# genpack-stencils

genpackアーティファクト作成のための参照部品集です。

## コンセプト

genpackにはアーティファクト間で構成要素を共有するインポート機能がないため、共通の設定や処理は各アーティファクトに複製して使います。このリポジトリはその際の**参照元**となる部品を、テーマ（機能単位）ごとにまとめたものです。

部品を直接ビルドに取り込む仕組みは提供しません。LLMがこのリポジトリを参照して適切な部品を見つけ、対象アーティファクトへの適用を支援することを前提にしています。

## ディレクトリ構成

テーマごとにディレクトリを設け、その下に通常のアーティファクトと同じ構造で部品を配置します。

```
<theme-name>/
  genpack.json5   # このテーマが追加・変更するキーのフラグメント
  files/
    build.d/      # ビルド時スクリプト
    ...           # その他の設定ファイル
  patches/        # 必要に応じて
```

## genpack.json5 フラグメントの書き方

各テーマの `genpack.json5` は**JSON5としてパース可能**でなければなりませんが、テーマに関係するキーだけを記述します（完全なアーティファクト設定ではありません）。

ファイル冒頭に以下のヘッダコメントを必ず記述してください：

```json5
// theme: <theme-name>
// <このテーマの概要説明>
// 依存: <前提となる他テーマ、またはなし>
// 競合するキー: <既存の genpack.json5 と衝突する可能性があるキー>
{
  // ...
}
```

`theme:` タグを統一しておくことで `rg 'theme:'` による全テーマ一覧の取得が可能です。

## アーティファクトへの適用ルール

テーマの部品をアーティファクトに取り込む際は、出典をコメントで残してください。

`files/build.d/` 内のスクリプトの場合：

```sh
# ref: <theme-name>/files/build.d/<script-name>
```

これにより、フラグメントに修正が生じた際に `rg 'ref: <theme-name>'` で影響範囲を特定できます。

## テーマの追加・更新

- 既存アーティファクトで確立した設定パターンをここに切り出してテーマ化する
- テーマ名はディレクトリ名から自明になるよう説明的につける（例: `locale-ja`, `systemd-networkd-dhcp`）
- フラグメントの変更時は、`rg 'ref: <theme-name>'` で取り込み済みのアーティファクトを洗い出して影響を確認する
